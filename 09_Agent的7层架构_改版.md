# Agent 的 7 层架构：一张图看懂工业级系统的全貌

> 《工业级 AI Agent 开发实战》系列 · 第 09 篇

学 Agent 最容易犯的错，是把所有问题都混在一起。

回答不准，说是模型不行；工具调错，说是 Prompt 没写好；用户越权看到数据，说是 RAG 出问题；循环停不下来，说是框架太差。

这些说法都可能对，也都可能错。

工业级系统里，最重要的不是先猜原因，而是先定位层级。

没有架构图，所有排查都是瞎猜。

## 先看 5 层

从功能视角看，一个 Agent 至少有 5 层。

第一层是模型层。负责发请求、收响应、处理 token、采样参数、流式输出、重试、限流和模型路由。第 02 篇讲的 LLM 调用，都在这一层。

第二层是记忆与检索层。负责短时上下文、长时记忆、RAG、权限过滤和 token 预算。Memory 和 RAG 在架构上其实是同一层：都是给无状态模型注入它本来没有的信息。

第三层是执行层。负责 Tool、Function Calling、Skill、MCP、幂等、参数校验和错误翻译。模型只做决策，真正动手的是这一层。

第四层是编排层。负责 Agent 循环、多 Agent、工作流、状态机、checkpoint、终止条件。ReAct、Plan-and-Execute、LangGraph、CrewAI 都主要在这一层。

第五层是交互层。负责用户怎么发问、文件怎么上传、流式怎么展示、人工审批怎么插入、微信/飞书/Slack 怎么接入。

这 5 层能让一个 Agent 跑起来。

但能跑，不等于敢上线。

## 工业级必须有第 6、第 7 层

第六层是安全与治理层。

它管的是：Prompt Injection、RBAC、PII 脱敏、工具权限、高危操作 HITL、审计日志。

没有这一层，Agent 可能把 system prompt 吐出来，可能越权查别人数据，可能被用户诱导调用危险工具。

企业级 Agent 没有第 6 层，不能上线。这不是加分项，是底线。

第七层是评估与反馈层。

它管的是：评测集、回归测试、trace、用户反馈、bad case 归因、灰度发布、版本回滚。

没有这一层，你改了一行 Prompt，都不知道效果是变好还是变差。模型换版本、RAG 改切块、工具描述调整，全部只能凭感觉。

第 7 层的核心价值，是让系统可衡量、可迭代、可防回退。

没有第 7 层的 Agent，不是没有 bug，是看不见 bug。

## 7 层架构的完整心智模型

L1 模型层：让模型说话。

L2 记忆与检索层：让模型知道该知道的东西。

L3 执行层：让模型的决策变成真实动作。

L4 编排层：让多个动作组成完整任务。

L5 交互层：让用户和系统稳定交互。

L6 安全治理层：让系统不越权、不泄密、不乱动。

L7 评估反馈层：让系统持续变好，并且知道自己有没有变差。

市面上大多数教程讲到 L4 就停了。框架也大多覆盖 L1-L4。

但真正区分玩具级和工业级的，是 L6 和 L7。

## 框架应该放在哪一层

OpenAI SDK 主要在 L1。它帮你调模型，不帮你做 Agent。

RAG 框架和向量库主要在 L2。它们解决知识注入，不解决循环编排。

MCP 在 L3。它标准化工具接入，不负责你的业务权限和幂等策略。

LangGraph、CrewAI、AutoGen 主要在 L4。它们解决编排范式，不自动补齐安全和评估。

Langfuse、LangSmith、RAGAS、Harness 这类工具主要在 L7。

Guardrails、OPA、权限网关、审计系统在 L6。

这张映射能避免一个常见误区："我用了 LangChain，所以我有完整 Agent 系统了。"

没有。你只是有了部分 L1-L4 的封装，L6 和 L7 仍然要自己建设。

## 出问题时怎么按层排查

接口 429、超时、流式中断，先看 L1。

回答引用了错误资料、记忆丢失、RAG 没召回，先看 L2。

工具选错、参数填错、副作用重复执行，先看 L3。

循环跑偏、多 Agent 互相拉扯、任务卡住，先看 L4。

用户前端看不到完整输出、审批状态不同步，先看 L5。

数据泄露、越权访问、Prompt Injection，先看 L6。

改了 Prompt 后效果说不清，线上质量波动无法定位，先看 L7。

工业级工程师和玩具级最大的区别，不是会更多框架，而是看到问题第一反应能说清：这是哪一层的问题。

## 收束

Agent 不是模型 + Prompt + 工具这么简单。

Agent 是一个 7 层系统。前 5 层决定能不能跑，后 2 层决定敢不敢上线。

架构能力的价值就在这里：它让复杂问题变成可定位、可拆解、可分工的工程任务。

没亲手写过一次，你对 Agent 的理解永远隔着一层框架。
